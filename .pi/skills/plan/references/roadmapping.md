## Roadmapping Lens

Use this reference when the work needs to be split into epics or sequenced.

Focus on:

- finding the smallest coherent set of epics that can deliver the intended outcome
- sequencing epics by dependency and delivery value
- merging epics that cannot be reviewed independently
- splitting epics that bundle unrelated deliverables
- making each epic's user-visible outcome, dependency boundary, and reason for existence explicit

Aim for a roadmap a stakeholder could understand without needing implementation details.
